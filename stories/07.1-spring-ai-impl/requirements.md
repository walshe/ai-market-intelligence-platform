# Requirement: Spring AI Implementation for LLM and Embeddings

## 1. Objective
Introduce a new `Spring AI` implementation for both LLM completions and vector embeddings as an alternative to the existing manual `RestClient`/`WebClient` implementations. This must be done without removing or breaking the current implementations.

## 2. Rationale
- **Standardization:** Leverage Spring AI's unified API for different AI providers.
- **Structured Output:** Utilize Spring AI's `BeanOutputConverter` and standard Jackson annotations (`@JsonPropertyDescription`, `@JsonClassDescription`) to handle JSON responses more reliably and with less manual prompt engineering.
- **Maintainability:** Reduce custom JSON parsing logic in `AnalysisServiceImpl`.

## 3. Functional Requirements

### 3.1. Spring AI Completion Client
- Implement `LLMCompletionClient` using Spring AI's `ChatModel`.
- Support both blocking (`complete`) and streaming (`streamCompletion`) operations.
- Extract usage metadata (input/output tokens, model name) from `ChatResponse` to maintain compatibility with `CostTrackingService`.
- Integration with `BeanOutputConverter` for structured JSON analysis results.

### 3.2. Spring AI Embedding Client
- Implement `EmbeddingClient` using Spring AI's `EmbeddingModel`.
- Support generating embeddings for text chunks.
- Extract usage metadata for cost tracking.

### 3.3. Configuration & Provider Selection
- Use `@ConditionalOnProperty(name = "application.llm.provider", havingValue = "spring-ai")` to activate the Spring AI beans.
- Ensure existing providers (`openai`, `bedrock`) continue to function when configured.
- Add necessary Spring AI starters to `pom.xml`.

## 4. Technical Specifications

### 4.1. Structured Output Model
Define a Record for the analysis response using Jackson annotations to describe fields to the LLM:
```java
public record AnalysisResponse(
    @JsonPropertyDescription("A concise summary of the financial analysis")
    String summary,
    
    @JsonPropertyDescription("A list of potential risk factors identified in the context")
    List<String> riskFactors,
    
    @JsonPropertyDescription("A score between 0 and 1 representing the assistant's confidence")
    double confidenceScore
) {}
```

### 4.2. Prompt Handling
- Shift the JSON schema definition from `application.yml` to the `BeanOutputConverter`.
- The `PromptBuilderService` should be updated or a new strategy used to include the format instructions generated by `converter.getFormat()`.

## 5. Constraints
- **Zero Regression:** Existing `OpenAICompletionClient` and `OpenAIEmbeddingClient` must remain fully operational.
- **Cost Tracking:** Every call through Spring AI must still result in a record in the cost tracking system.
- **No Heavy NLP:** Maintain the "Scope Discipline" (Rule 17.1) - do not introduce heavy NLP libraries or additional vector databases.
